Used technologies on frontend

On the frontend we have decided for Angular framework because of our love for clean architecture that this framework forces it on developers. We like to have order of folder structure and clean config file putted everything in one place. Angular used with Angular Cli is very powerfull tool for managing of reusable components and bindig logic with views. Naturally with Angular came Typescript language, but even not we would choose it anyway. We love stron typing for it safety and readability for other programmers. It provides static checking, so programming become with it really easy thing and code becomes even more bugproof. The same reasons as above standing when we have choosen Sass as our precompilation language for css. It develops productivity and allow us to use variables, so with little programming changes we have impact on whole site and it it also free of bugs. Out choises we could simply reassumed by DRY principle. :)

It is deployed on

Application was deployed on Azure Portal as a App Service for API (backend part) and on the Blob Storage as a static web page (output of Angular build). Database is in pay-as-you-go service mode, sql server. (Link for api) Static webpage simply calls this API which send proper request to database.
